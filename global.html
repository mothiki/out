showMonitoring(id, elementID)

Function to start Spectrometer monitoring threads This function calls the controller startMonitoringServices with a unique id to maintain session. The controller starts multiple threads that return values as Java MessageSender Events and based on the returned String, we separate them into different monitoring parameters. Returned values from MessageSender is of the form "XX|ABC|DEF ..." where XX is an arbitrary number that maps to the parameter being monitored and the rest of '|' pipe separated values are different values corresponding to that parameter The current parameter mappings for XX are: 00: Elapsed Time 01: Laser Temperature in C 02: TEC Temperature in C 03: Laser Enable 10: CaF2 Peak Photon Count 06: PD Power (Experimental) (Displays in Console for Now)
Parameters:
Name Type Description
id string A unique ID that can be used to establish a session between showMonitoring and the EventSource from startMonitoringServices
elementID string The suffix to the HTMLElements where the parameters are to be displayed
